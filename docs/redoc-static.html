API for Inlet.Shop.

Vendor

Operations related to vendors

Create password

Allows a new user to create their password in order to grant them access.

path Parameters
vendor_id
required
string
Example: 68e437e1f2235a39d7ce5f47

Vendor ID (ObjectId)

token
required
string
Example: 92d98d7b-1279-488e-ae2d-652920c06209

Vendor token (UUID)

Request Body schema: application/json
password
required
string <password>

New password for the vendor

confirmPassword
required
string <password>

Same password for confirmation

Responses

Production server

https://api.inlet.shop/vendor/{vendor_id}/password/{token}

Development server

http://127.0.0.1:8001/vendor/{vendor_id}/password/{token}

Request samples

  • Payload
Content type
application/json
{
  • "password": "password123",
  • "confirmPassword": "password123"
}

  • 400

  • 404
  • 500
Content type
application/json
{
  • "success": true
}

VendorAuth

boolean

Determines whether the vendor has completed the onboarding process for Stripe.

new_order_send_email
boolean

Whether the vendor is sent an email whenever a new order is created.
